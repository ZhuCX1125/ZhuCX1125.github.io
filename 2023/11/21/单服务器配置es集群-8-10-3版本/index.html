启动单节点集群

搭建使用es 8.10.4版本，整体配置流程如下

  1. 安装 Docker。访问获取 Docker为您的环境安装 Docker。
    如果使用 Docker Desktop，请确保分配至少 4GB 内存。您可以通过转到 “设置”>“资源” 来调整 Docker Desktop 中的内存使用情况。
  2. 创建一个新的 Docker 网络。
  1. 拉取 Elasticsearch Docker 镜像
  1. 为您的环境安装 Cosign。然后使用 Cosign 验证 Elasticsearch 镜像的签名。

  1. 启动 Elasticsearch 容器

使用 -m 标志设置容器的内存限制。这样就无需手动设置 JVM 大小。

  1. 复制生成的弹性密码和注册令牌。这些凭据仅在您首次启动 Elasticsearch 时显示。您可以使用以下命令重新生成凭据。

  1. 对 Elasticsearch 进行 REST API 调用以确保 Elasticsearch 容器正在运行。

添加更多节点

  1. 使用现有节点为新节点生成注册令牌。
  • 这里运行的时候也必须是root用户
  • 注册令牌的有效期为 30 分钟。
  1. 启动一个新的 Elasticsearch 容器。包含注册令牌作为环境变量。
  1. 调用 cat 节点 API 来验证节点是否已添加到集群中。
